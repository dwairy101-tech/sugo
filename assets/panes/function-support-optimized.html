Function / Host Support Optimized

Use this page for feature-related issues, host tools, live functions, agency joining, and host workflow cases.

Required information before checking

  • User ID / host ID.
  • Feature name or page path.
  • Screenshot or screen recording.
  • Device model, OS, and app version.
  • Exact time the issue occurred.
  • Whether the issue happens to one user or multiple users.

CS decision flow

  1. Confirm the feature or workflow involved.
  2. Ask for a screen recording if the issue is visual or step-based.
  3. Check whether the user meets feature eligibility requirements.
  4. If the issue is policy-related, use SOP Only mode.
  5. If it looks technical, collect device/app details and escalate to the relevant technical or operations team.

Escalation

  • Feature missing despite eligibility.
  • Repeated system error.
  • Host/agency application issue not explained by normal rejection reasons.
  • Live or video function not working after standard troubleshooting.

Customer reply template

Welcome to the SUGO family!

We sincerely apologize for the issue you are experiencing.

To help us check this feature issue, please provide your user ID, the feature name, a screenshot or screen recording, your device model, app version, and the exact time the issue occurred.

We will review the issue according to the relevant support procedure.

Thank you.

Function / Host Support Optimized

استخدم هذه الصفحة لمشاكل المزايا، أدوات المذيعات، وظائف البث، الانضمام للوكالة، وحالات سير عمل المذيعات.

المعلومات المطلوبة قبل الفحص

  • رقم المستخدم أو رقم المذيعة.
  • اسم الميزة أو مسار الصفحة.
  • لقطة شاشة أو تسجيل شاشة.
  • نوع الجهاز ونظام التشغيل وإصدار التطبيق.
  • الوقت الدقيق لحدوث المشكلة.
  • هل المشكلة لدى مستخدم واحد أم عدة مستخدمين.

خطوات قرار خدمة العملاء

  1. تحديد الميزة أو سير العمل المتعلق بالحالة.
  2. طلب تسجيل شاشة إذا كانت المشكلة مرئية أو مرتبطة بخطوات.
  3. التحقق مما إذا كان المستخدم يستوفي شروط استخدام الميزة.
  4. إذا كانت الحالة مرتبطة بسياسة، يتم استخدام وضع SOP Only.
  5. إذا بدت مشكلة تقنية، يتم جمع بيانات الجهاز والتطبيق والتصعيد للفريق المختص.

التصعيد

  • الميزة غير ظاهرة رغم استيفاء الشروط.
  • خطأ نظام متكرر.
  • مشكلة طلب مذيعة/وكالة لا تفسرها أسباب الرفض الطبيعية.
  • مشكلة في البث أو الفيديو بعد تجربة خطوات الاستكشاف الأساسية.

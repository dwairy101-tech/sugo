# فحص Cloudflare الآمن لمشروع SUGO

شغّل الملف التالي من جذر المشروع:

```text
CHECK_CLOUDFLARE_ACCESS.bat
```

## ماذا يفعل؟

- يفتح تسجيل دخول Cloudflare الرسمي عبر OAuth عند الحاجة.
- يقرأ اسم الـWorker وحالة آخر Deployment والنسخ المنشورة.
- يعرض **أسماء** الـSecrets وأنواعها فقط، ولا يقرأ قيمها.
- يفحص KV وملف `wrangler.toml` ويبني الـWorker محلياً باستخدام `--dry-run` دون نشره.
- يفحص `/health` وحماية `/diagnostics` وCORS وواجهات المحتوى والصور.
- يرسل طلب AI تشخيصياً صغيراً لا يحتوي أي بيانات شخصية.
- ينشئ ملفاً باسم قريب من:

```text
SUGO_CLOUDFLARE_REPORT_20260712_120000.zip
```

ارفع ملف التقرير المضغوط إلى المحادثة حتى تتم مراجعة إعدادات Cloudflare الفعلية.

## الأمان

الأداة في وضع **قراءة فقط** ولا تنفذ `wrangler deploy` الحقيقي، ولا تعدّل Secrets أو KV أو Routes. تسجيل الدخول يتم في موقع Cloudflare عبر المتصفح، ولا يجب لصق Token أو كلمة مرور داخل نافذة PowerShell.
